# System-Architecture-using-UML
System design and UML modelling for a interactive game Design Dash
Project Overview:
Architecture Type: Client-Server Web Application
Core Logic: Real rime synchronized session management with Host, Registered User, and Guest User privileges.
Design Focus: Scalable database structure, secure session handling (QR & Link), and real-time state management for ongoing games.
UML Modeling & System Diagrams:
Use Case Diagram: Mapping actor interactions system boundaries, and permission levels.
Class Diagram: Detailed object oriented structure including attributes, methods, inheritance, and multiplicities.
Sequence Diagram: Time focused message flow between objects during game initialization and gameplay loops.
Activity Diagram: Step by step workflow of the system from session creation to game completion.
State Machine Diagram: Lifecycle of the core game state, including transitions and event triggers.
